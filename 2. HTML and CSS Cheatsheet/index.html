Basic Sytax

Elements

An element consists of content between an opening and closing tags (see Figure 1 below).

Elements can be either block-level or inline-level. An block element is visually separate from the surrounding elements. An inline element can be placed within block elements without disrupting the flow of the page. Let's analyze this page to explain the difference. Notice how this block of text is separated from the text above and below it. This is the case because each paragraph is written in a separate p element, which is a block element. However, within this paragraph, the words block-level and inline-level are italicized. This was done by placing each word within the em tag, an inline element.

Some tags, such as br / are self closing and do not require a closing tag. These are known as void elements (br / and br are both valid).

Attributes

Many tags also have attributes where the developer can specify a value (see Figure 2). For example, the image tag img / has a required attribute "src" (source). The associated value is a link to the image to display. This is an example of a required attribute, but many are optional.

Global Attributes

There are two commonly used global attributes. These serve a very important purpose when styling a website with CSS.

  • iduniquely identifies a tag (can only be applied to 1 tag)
  • classidentifies multiple tags (can be applied to unlimited tags)

Tables

A table is initialized with a table element. The contents of this tag are split into 3 sections: thead (table head), tbody (table body), and tfoot (table foot).

The thead element contains a single tr (table row). Within this row, is a th (table header) representing each column in the table.

The tbody contains each of the rows (tr). Each row contains a td (table data) corresponding with each column.

The tfoot is optional and contains a final row. For exmple, this is useful if the table has a row displaying the sum of each column.

Lists

Lists can be of 3 types of lists: ul (unordered lists), ol (ordered lists), and dl (description lists).

The first two contain a li (list item) element for each item of the list. The description lists contain a dt (description term) and dd (description details) elements for each item in the list.

Page Structure

Elements are organized in a hierarchical structure and distinguished by level of indentation. Indentation is commonly either 2 or 4 spaces.

Each HTML document must have the statement, <!DOCTYPE HTML> at the top. Additionally, everything must be within an html element.

An html file will genrally be split into 2 parts: head and body. The head element contains critical information such as the title of the website and links to the CSS document. Within the body, there are usually 3 sections: header, main, and footer. The header contains information at the top of the document such as navigation links or the title. The body contains the main section of the document. The footer contains information at the bottom of the pages such as contact details.

Semantic HTML

Using the common tags listed above, one has the resources to create a fully functinoal html document. However, it can be useful to use more descriptive tags. This can make the website more appealing to search engines and improve the experienced for those using screen readers. Below are some of the commonly used semantic HTML tags. Note that these provide minimal, if any new functionality when compared to the above tags. It is, however, good practice to use these when applicable and only use the generic tags such as div as a last resort.
